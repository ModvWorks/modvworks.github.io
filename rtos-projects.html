// what's inside

Five layers of the RTOS stack

01 /
Bare Metal Scheduling
01_bare_metal/

Seven applications built twice — once with pure cooperative round-robin (no interrupts), once with ISR-driven task signaling. The mechanics FreeRTOS abstracts away are fully visible here. Starvation, priority inversion, context-switch cost: all on display, no safety net.

STM32F4 round-robin ISR cooperative

02 /
FreeRTOS Fundamentals
02_freertos/

Production-oriented examples for every major FreeRTOS primitive. Separate task files, shared config header, HAL integration — and at least one deliberate failure mode shown and explained per module. Stack sizing, heap tradeoffs, ISR-safe variants all covered.

FreeRTOS v10+ xQueueSend xSemaphore xTimerCreate heap_4
browse →
03 /
Industrial Applications
03_industrial_applications/

110+ RTOS reference implementations across 12 industry domains. Each application is grounded in real industrial constraints: deterministic timing budgets, fault containment, sensor fusion, and communication protocol integration. Task maps and timing analysis included.

04 /
Advanced RTOS Patterns
04_advanced_patterns/

Design patterns adapted for embedded production environments: Active Object, Producer-Consumer with DMA handoff, Monitor Object, and the Reactor pattern. Built for constrained memory, hard real-time deadlines, and interrupt-driven execution — not academic exercises.

Browser-based interactive RTOS scheduling simulator — no build required. Visualizes RMS, EDF, priority preemption, task blocking, and CPU utilization in real time. An extracted module from the Embedded Control Systems Studio (ECSS).

// hardware targets

Written for real boards,
not simulators.

All embedded code is written and tested on physical hardware. The STM32F407 Discovery is the primary target — Cortex-M4 with FPU, 1MB flash, 192KB RAM, enough headroom to make scheduling decisions visible without hiding behind infinite resources. Porting to other Cortex-M targets is straightforward — the concepts translate directly even when peripheral init differs.

Toolchain: STM32CubeIDE + STM32CubeMX · FreeRTOS v10.4+ · ARM GCC
